Versionsstyring

Versionsstyring er en metodisk tilgang til kodning og programmering, som kan have til formål at holde styr på bl.a. hvilke ændringer der har været i koden, hvornår det blev ændret, delprojekter, for at nemmere kunne samarbejde osv.

GitHub

GitHub er kort sagt en webplatform, hvorpå man kan oprette Git-repositories. Man kan definere sitet som værende et socialt medie, i det man kan kontakte hinanden og dele kode. GitHub er et meget populært site, og bliver ofte brugt som portfolio for kode. Derfor er der også mange arbejdsgivere, som efterhånden forventer at man kan fremvise sin portfolio via GitHub. Programmet kan både åbnes gennem en browser, og via deres officielle desktopprogram. Mange Open Source projekter programmer hostes her, og kan også være meget nyttig hvis man mangler et sted at hente både store og små stumper af kode til egne projekter.

Git

Git er et versionsstyrringssystem, oprindeligt skrevet af Linus Thorvalds, for at holde styr på udviklingen af Linux efterhånden, som flere kom med.

GitHub begreber

Repository (Repo)

En samling af filer, som en projektmappe.


Branch

En forgrening, eller ”parallel” version i dit repo, som ikke berør din master.


Commit

En revision som at gemme, men med mulighed for at finde tilbage.


Pull Request

Et forslag til en ny udgave (rettelser).


Diff

Diff er forskellen mellem to commits.


Merge

Et merge er når en ændring i en branch samles med en anden (eks. gennem en pull request).


Pull

Når du henter ændringer og merger dem.
